# 04 Content Style

## Voice
- Write in factual, audit-friendly language.
- Avoid hype terms.
- Prefer concrete system behavior statements.

## Preferred Phrasing
- "research-first"
- "governance-first"
- "non-neural machine learning"
- "proposal-to-commit"
- "deterministic replay evidence"
- "public-safe synthetic data"

## Naming Conventions
- Public nav label: `Demo` (not "Replay Demo").
- Use `Research Dashboard`, `Governance Replay`, `Model Exams` as proper names.
- Use "under construction" for paused surfaces.

## SEO Guidance
Each page should include:
- A specific page description mentioning one concrete Elora capability.
- `og:description` aligned with the same capability.
- `twitter:description` shortened but consistent.

## Do Not
- Claim production behavior for demo-only screens.
- Expose internal-only telemetry labels not meant for public release.
- Use ambiguous terms like "AI magic" or "fully autonomous authority".

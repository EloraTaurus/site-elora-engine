# 05 Implementation Checklist

Use this before shipping visual/content changes.

## Colors
- [ ] Uses canonical tokens from `02-color-tokens.md`.
- [ ] Body/data text contrast is readable on white background.
- [ ] No one-off hex values for core controls unless justified.

## Components
- [ ] Header readable at top and on scroll.
- [ ] Buttons follow primary/secondary/disabled pattern.
- [ ] Cards, tables, and pills match shared style.
- [ ] Tour popup stays white and readable.

## Content
- [ ] Copy reflects research-first + governance-first positioning.
- [ ] Non-neural behavior detection language included where relevant.
- [ ] Demo pages clearly state synthetic/public-safe boundaries.

## SEO + Structure
- [ ] `description`, `og:description`, `twitter:description` updated.
- [ ] Canonical URL is correct.
- [ ] Navigation includes current top-level pages.
- [ ] `sitemap.xml` updated if a new page is added.

## Quick QA
- [ ] Desktop: hero, nav, cards, and footer look consistent.
- [ ] Mobile: nav toggle, spacing, and button wrapping are clean.
- [ ] No broken links to paused pages.

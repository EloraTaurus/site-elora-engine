Security‑first design

  • Default‑deny tools and web access unless explicitly allowed.
  • Signed, time‑bound requests with origin allowlists.
  • Strict separation of public status and privileged admin data.
  • Audit logging for logins, config changes, and reflections.
  • Safe fallbacks when trusted sources aren’t available.
  • Short‑lived sessions to reduce exposure and leakage.
